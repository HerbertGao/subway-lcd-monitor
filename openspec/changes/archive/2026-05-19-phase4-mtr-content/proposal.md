## 为什么

港铁风重制期一（视觉风格）、期二（运行状态）已完成并归档，渲染缺陷亦已由 `fix-mtr-rendering-bugs` 修复。但与港铁实拍参考（见 `docs/mtr-ui-reference.md`，基于西铁线实拍视频整理的界面参考）对照，当前实现仍有明显差距：

- **换乘信息完全缺失** —— 线路图场景不渲染换乘线路标签（期一曾明令禁止、留待后续阶段）；全程线路图未承载换乘网络信息（实拍中全程线路图在临近换乘站时即展示换乘线路网络）。
- **站名特写若干细节不符** —— 黄色安全条缺少港铁式两端警示图标、版式不随文案变化；开门方向提示样式与实拍不一致。

期三补齐这些换乘相关内容与界面细节，使 LCD 报站屏更贴合港铁实拍。沿用既有北京地铁数据（燕房线、房山线，经阎村东互为换乘），不引入新城市数据。

## 变更内容

本期为渲染层增量，分 5 块：

1. **换乘线路标签** —— 全程线路图、近段线路图场景的换乘站圆点下方渲染换乘线路标签（港铁式彩色线路名牌：换乘线路自身色 + 线路名，取自 `transfers.lineName`）。换乘数据取自线路 JSON 的 `station.transfers` 字段（含 `lineId` / `lineName` / `lineColor`，无英文名字段，故标签标注中文 `lineName`）。**BREAKING**（对 spec）：解除期一「不渲染换乘线路标签」的限制。无换乘的站不渲染标签。

2. **站名特写黄条细节** —— 站名特写黄色安全条：「請小心空隙」版本在黄条两端各渲染一个港铁式红圈警示图标（纯 SVG / CSS 绘制），「請勿靠近車門」版本不带图标；黄条版式随文案微调。

3. **开门方向提示本侧闪烁** —— 本侧开门（`doorSide` 为 `right` 或 `both`）的开门方向提示，由当前的**恒定**绿底圆角胶囊改为在「绿底圆角胶囊↔黑字无底」两态间**闪烁**（周期约 2s）；对侧开门（`left`）保持静态黑字无底。开门提示为中英双语两行（中文行 + 英文小字行）。`doorSide` 三态与空值渲染逻辑、各态文案语义不变。

4. **全程线路图换乘支线** —— 下一站为换乘站时（`RUNNING` 状态、`nextStation` 的 `transfers` 含至少一个可在已加载线路中解析到的换乘线路），全程线路图在该换乘站处把换乘线路作为一条**支线**画出 —— 换乘线路自身色的短线路条 + 站点，以换乘节点连接当前线路的换乘站；底部蓝色提示条同时切换为换乘提示文案（「往房山线各站，请在此站换乘」中英双语）。换乘网络信息直接融入全程线路图、**不**新增独立的「全网换乘图」场景 —— 与港铁实拍一致（实拍中全程线路图本身承载换乘网络信息）。下一站非换乘站时全程线路图照常显示、无支线。

5. **方向箭头位置与闪烁修正** —— 线路图场景的行进方向箭头：位置由「紧贴当前站圆点右侧」改为「最后一个已过站与下一站之间的线路段上」；并由静态黑色三角改为与下一站圆点**同步闪烁** —— 在绿色实心与白色实心两态间交替、描边恒深色。

## 功能 (Capabilities)

### 新增功能

（无 —— 本期不新增能力。换乘标签与换乘支线作为既有 `mtr-visual-style` 线路图能力的增量需求。）

### 修改功能

- `mtr-visual-style`: 移除「期一不渲染换乘线路标签」需求；新增「线路图场景换乘站必须渲染换乘线路标签」「全程线路图下一站为换乘站时必须渲染换乘线路支线」两条需求；修改「站名特写场景必须采用港铁版式」需求（补充黄条两端警示图标与随文案变化的版式）、「线路图场景必须采用港铁版式并含底部提示条」需求（行进方向箭头位置与闪烁交由 `mtr-train-states` 定义）。
- `mtr-train-states`: 修改「站名特写必须显示开门方向提示」需求（本侧开门提示在「绿底胶囊↔黑字」间闪烁、对侧静态）；新增「线路图方向箭头必须按运行状态定位并与下一站圆点同步闪烁」需求。

## 影响

- **渲染层**：
  - `src/components/lcd/scenes/FullRouteScene.vue` —— 换乘标签、方向箭头、换乘线路支线与条件蓝条文案
  - `src/components/lcd/scenes/NearbyScene.vue` —— 换乘标签、方向箭头
  - `src/components/lcd/scenes/useRouteLayout.ts` —— 换乘标签布局、方向箭头定位（`directionArrow`）
  - `src/components/lcd/scenes/ArrivalScene.vue` —— 黄条警示图标与版式、开门提示样式
  - `src/components/lcd/LcdScreen.vue` —— 不新增场景映射（移除曾新增的 `network-map` 注册）
- **核心逻辑层**：`src/core/local-network.ts` —— 换乘线路解析纯函数（供全程线路图换乘支线使用，可脱离 Vue 单测）；`train-state-visuals.ts` —— 黄条图标判定、换乘提示文案。
- **数据层**：沿用现有 `line-yanfang.json` / `line-fangshan.json` / `network.json`，不新增数据文件。
- **依赖**：不引入第三方依赖或图片资源（图标纯 SVG / CSS 绘制）。
- **规格**：修改 `mtr-visual-style`、`mtr-train-states`，不新增 capability。

## 非目标

- **不新增独立的「全网换乘图」场景** —— 换乘网络信息融入全程线路图。
- 不引入新城市数据、不扩充燕房线 / 房山线之外的线路。
- 不引入第三方依赖或图片资源。
- 不改 LCD 屏 7∶2 比例与既有三场景的布局骨架。
- 不做蓝条文案的多主题轮换扩充。
- 不改换乘数据结构（沿用既有 `transfers` 字段）。
- 不在近段线路图渲染换乘支线（站点范围窄、支线意义有限；换乘信息在近段图由换乘标签体现）。
