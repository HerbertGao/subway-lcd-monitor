## 新增需求

### 需求:线路图方向箭头必须按运行状态定位并与下一站圆点同步闪烁

线路图场景的行进方向箭头必须位于「最后一个已过站与列车正驶向的下一站之间」的线路段上（约线段中部），实心三角指向行进方向。

方向箭头必须与下一站圆点同步闪烁——在与下一站圆点相同的闪烁周期与相位下，于「绿色实心」与「白色实心」两态之间交替；箭头描边恒为深色、不随闪烁改变。箭头的填充色（绿色、白色）与描边色必须经主题系统的 `ColorConfig` 与 CSS 变量提供，不在组件中写死。

#### 场景:方向箭头位置
- **当** 线路图场景渲染行进方向箭头
- **那么** 箭头位于最后一个已过站与下一站之间的线路段上、实心三角指向行进方向

#### 场景:方向箭头与下一站圆点同步闪烁
- **当** 线路图场景渲染且存在列车正驶向的下一站
- **那么** 方向箭头在绿色实心与白色实心两态间闪烁，其闪烁周期与相位同下一站圆点严格一致（箭头绿色态对应下一站圆点黄色态、箭头白色态对应其白色态），箭头描边恒为深色

## 修改需求

### 需求:站名特写必须显示开门方向提示

站名特写必须根据展示站的 `doorSide` 显示港铁式开门方向提示（中英双语，呈中文行与英文小字行两行排布）；`doorSide` 的 `left`、`right`、`both` 三态与空值均须有明确处理，各态的提示文案语义不变。

开门方向提示的呈现样式必须按本侧 / 对侧区分：本侧开门（`doorSide` 为 `right` 或 `both` —— 列车本侧车门会开启）的开门方向提示必须闪烁——在「绿底圆角胶囊」与「黑字无底」两态之间交替；对侧开门（`doorSide` 为 `left`，「請往另一邊落車」类）的开门方向提示为静态、不闪烁。

#### 场景:右侧开门
- **当** 展示站 `doorSide` 为 `right`
- **那么** 显示「請在這邊落車」类港铁式开门方向提示，且该提示在「绿底圆角胶囊」与「黑字无底」两态间闪烁交替

#### 场景:左侧开门
- **当** 展示站 `doorSide` 为 `left`
- **那么** 显示「請往另一邊落車」类港铁式开门方向提示，该提示为静态黑字、不闪烁

#### 场景:双侧开门
- **当** 展示站 `doorSide` 为 `both`
- **那么** 显示双侧开门的港铁式提示（文案表两侧车门均会开启、文案语义不变），且因含本侧车门开启、该提示按本侧规则在「绿底圆角胶囊」与「黑字无底」两态间闪烁交替

#### 场景:开门方向数据缺失
- **当** 展示站无 `doorSide` 数据
- **那么** 不显示开门方向提示，站名特写其余内容正常渲染
