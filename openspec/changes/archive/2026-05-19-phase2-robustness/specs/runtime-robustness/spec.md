## 新增需求

### 需求:必须提供分级日志设施

项目必须提供一个分级 logger，至少支持 `debug`、`info`、`warn`、`error` 四个级别，按当前生效级别阈值过滤输出。该 logger 必须为纯 TypeScript 实现、不依赖 Vue 运行时，可被核心逻辑层、状态管理与组件统一调用。

#### 场景:按级别阈值过滤
- **当** 生效日志级别为 `warn`
- **那么** `debug` 与 `info` 调用不产生输出，`warn` 与 `error` 调用产生输出

#### 场景:核心逻辑层可调用 logger
- **当** 在不依赖 Vue 运行时的核心逻辑模块中导入并调用 logger
- **那么** 调用正常工作，且该模块不引入对 `vue` 的运行时依赖

### 需求:数据校验必须能区分重复站点 ID 与重复线路 ID

数据校验器产出的校验问题必须用不同的类型标识区分「重复站点 ID」「重复线路 ID」「无效换乘线路引用」三类问题，禁止把重复线路 ID 标识为重复站点 ID 的类型。

#### 场景:线网含重复线路 ID
- **当** 线网数据中存在重复的线路 ID
- **那么** 校验产出的问题类型标识为「重复线路 ID」专属类型，与「重复站点 ID」类型相区别

#### 场景:线路含重复站点 ID
- **当** 某线路数据中存在重复的站点 ID
- **那么** 校验产出的问题类型标识为「重复站点 ID」类型

### 需求:线网数据加载后必须执行完整性校验

加载某城市的线网与线路数据成功后，必须调用数据校验器执行完整性校验。校验发现的问题（重复站点 ID、重复线路 ID、无效换乘线路引用）必须经 logger 以告警级别输出，并记入状态管理中可供 UI 查询的校验结果。校验问题视为非致命，不得阻断线路数据的加载。

#### 场景:数据存在校验问题
- **当** 加载的线网数据存在重复站点 ID、重复线路 ID 或无效换乘引用
- **那么** 校验问题经 logger 告警输出、记入可查询的校验结果状态，且线路数据仍正常加载

#### 场景:数据无校验问题
- **当** 加载的线网数据通过完整性校验
- **那么** 校验结果状态为空，不产生告警输出

### 需求:数据加载失败必须被兜底而不导致应用崩溃

当 `loadNetwork` 或 `loadLine` 因数据文件缺失而抛出异常时，状态管理层必须捕获该异常，将其转化为可呈现的、带定位上下文（操作、城市、可选线路）的错误状态，并经 logger 记录；异常禁止向上冒泡导致应用崩溃。加载过程必须为事务式——加载失败时禁止改变状态管理中已有的有效数据（城市、线网、线路、可用线路列表）。

#### 场景:选择数据文件缺失的城市或线路
- **当** 用户选择一个数据文件缺失的城市或线路
- **那么** 异常被捕获、转为带定位上下文的错误状态并经 logger 记录，应用不崩溃

#### 场景:城市的某线路数据文件缺失
- **当** 用户选择的城市线网文件存在、但其引用的某条线路数据文件缺失
- **那么** 错误状态标明失败操作为线路加载、并包含具体的城市与线路标识，应用不崩溃

#### 场景:加载失败不污染既有状态
- **当** 在已成功加载某城市后，再选择一个数据文件缺失的城市
- **那么** 错误状态被设置，且先前已加载的城市、线网与线路数据保持不变、未被污染

#### 场景:加载失败后再成功加载会清空错误状态
- **当** 一次加载失败设置了错误状态后，再次选择并成功加载一个有效的城市
- **那么** 错误状态被清空为无错误

### 需求:应用必须提供错误边界以避免白屏

应用必须提供组件级错误边界，捕获其子树在渲染或生命周期中抛出的未捕获异常，并渲染降级 UI 而非白屏。降级 UI 提供的「重试」必须能真正使子树重新挂载、而非仅隐藏错误界面。应用入口必须注册全局错误处理器，对未被组件边界捕获的异常经 logger 以错误级别记录。

#### 场景:子树组件抛出渲染异常
- **当** 被错误边界包裹的子树中某组件在渲染或生命周期中抛出未捕获异常
- **那么** 错误边界捕获该异常、渲染降级 UI，应用不出现白屏

#### 场景:错误边界记录被捕获的异常
- **当** 错误边界捕获到子树抛出的异常
- **那么** 该异常经 logger 以错误级别记录（错误边界不依赖全局错误处理器来记录被它捕获的异常）

#### 场景:从降级 UI 重试
- **当** 在降级 UI 上触发「重试」
- **那么** 错误边界清除其错误状态并使子树重新挂载，子树从初始状态重新渲染

#### 场景:全局错误处理器记录异常
- **当** 出现未被组件边界捕获的异常
- **那么** 全局错误处理器经 logger 以错误级别记录该异常

### 需求:应用卸载时必须清理模拟资源

根组件卸载时，必须调用模拟状态仓库的 `destroy()`，以清理列车状态机与场景轮播持有的定时器，避免资源泄漏。

#### 场景:根组件卸载
- **当** 根组件被卸载
- **那么** 模拟状态仓库的 `destroy()` 被调用，列车状态机与场景轮播的定时器被清除
