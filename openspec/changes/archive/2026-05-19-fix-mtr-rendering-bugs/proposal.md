## 为什么

期四（港铁化视觉 + 细腻状态）归档后，用 Playwright 基于燕房线（9 站）与房山线（16 站）对整体功能做了一轮端到端测试，并与港铁实拍参考帧对照，发现 6 个渲染缺陷——其中 1 个为 P0（运行中线路图整屏空白，应用核心画面不可用）。这些缺陷均属期四实现遗留、未被既有单元测试覆盖（涉及 Vue scoped 编译行为、SVG 等比缩放、密集线路布局等运行时表现）。本变更集中修复这些缺陷，不引入新功能。

## 变更内容

修复 6 个渲染缺陷，全部位于渲染层：

1. **【P0】场景切换过渡失效、运行中线路图整屏空白。** `LcdScreen.vue` 的场景切换过渡 CSS（`.lcd-fade-*` / `.lcd-slide-*`）原写在 `<style scoped>` 内，被编译为带 `[data-v-x]` 属性的选择器；而 `<Transition>` 的过渡 class 加在 `<component :is>` 动态渲染的子场景根元素上，该元素只带子组件自身的 scoped 属性、不带 `LcdScreen` 的 `data-v` 属性 —— scoped 选择器永远命中不了，过渡 `transitionDuration` 退化为 `0s`、过渡完全失效；叠加 `<Transition mode="out-in">` 后会卡死在渲染 `<!---->`（`currentSceneComponent` 有效但场景始终不挂载），导致 `RUNNING` 状态下线路图整屏空白。修复：过渡 CSS 移出 scoped；`.lcd-screen` 作为定位上下文、场景根元素绝对定位重叠，使 cross-fade 期间新旧场景叠放而非作为 flex 兄弟并排闪现；不使用 `mode="out-in"`。

2. **线路图场景未填满 LCD 屏、底部蓝条不贴屏底。** 全程线路图 SVG 采用固定高度 viewBox（宽随站点数增长），等比缩放后 viewBox 宽高比与 LCD 屏 7∶2 不一致，站点越多 SVG 内容越扁、上下留白越大，底部蓝色提示条悬在屏中部。修复：让线路图场景填满整个 LCD 屏高度、蓝色提示条贴 LCD 屏底部。

3. **密集线路下英文站名横向重叠。** 站点多且英文名较长时（房山线「良乡大学城北 Liangxiang Daxuecheng Bei」一带），相邻站英文名在站间距内重叠糊成一团。修复：让站名在站间距内自适应（字号收缩 / 长名压缩），保证密集线路下站名不重叠、仍可读。

4. **页面外层背景未港铁化。** App 外层页面背景仍为深蓝 `#1a1a2e`，与港铁化的浅色 LCD 屏视觉不协调。修复：将页面外层背景调整为与港铁风格协调的中性深灰；页面外壳背景属应用外壳、不属 LCD 屏主题范畴，色值在全局样式中集中以单一 CSS 变量承载，不散落硬编码于多个组件（不纳入主题系统 `ColorConfig`）。

5. **线路图下一站闪烁圆点配色错误。** 当前线路图「正驶向的下一站」圆点的闪烁动画在「黄色实心」与「白色空心圆环（填充屏底色、白色描边）」两态间交替。经与港铁实拍闪烁采样帧（`/tmp/mtr-flash/`）逐帧比对，真实港铁该圆点闪烁是在「黄色实心」与「白色实心」两态间交替，且圆点描边在闪烁全程恒为深色（黑色）不变、不出现空心圆环、描边颜色不变。修复 `FullRouteScene` / `NearbyScene` 的下一站圆点闪烁动画：填充在黄↔白实心间交替，描边恒定深色。

6. **站名特写圆点未随运行状态闪烁。** `ArrivalScene` 的圆点当前为静态白色实心。经闪烁帧比对，站名特写在 `ARRIVING` 状态下其展示站（即正驶向的下一站）圆点应与线路图下一站圆点同规律闪烁——在黄色实心与白色实心之间交替、描边恒为深色不变；`STOPPED` / `DEPARTING`（展示站为已停靠的当前站）时圆点为白色实心、不闪烁。修复 `ArrivalScene`：按 `trainState` 驱动圆点的闪烁 / 静止两态。

另：测试中曾疑站名特写圆点偏空心，经核查其为 `background: var(--lcd-station-dot)` 白色实心圆点（白圆叠在浅灰屏底上对比偏低属正常观感），圆点形态本身正确；Bug 6 仅补充其随运行状态的闪烁行为。

## 功能 (Capabilities)

### 新增功能

（无 —— 本变更为纯缺陷修复，不引入新能力。）

### 修改功能

- `mtr-visual-style`: 澄清线路图场景的港铁版式要求 —— 线路图场景须填满整个 LCD 屏、底部蓝色提示条贴 LCD 屏底；且密集线路（站点较多）下站名不得重叠。并放宽「站名特写场景」需求中圆点的颜色描述——圆点的状态化着色交由 `mtr-train-states` 定义。均为对既有需求的澄清 / 补强，不新增能力。
- `mtr-train-states`: 修正「线路图站点圆点必须按列车运行状态着色」需求 —— 下一站圆点闪烁是在「黄色实心」与「白色实心」之间交替、描边恒为深色不变，取代原「黄色实心与白色空心之间交替」的错误描述。并新增「站名特写圆点必须按运行状态着色与闪烁」需求 —— `ARRIVING` 时圆点闪烁黄↔白实心、`STOPPED` / `DEPARTING` 时静态白实心。

（Bug 1 的场景切换过渡修复属纯实现细节，不改变任何 spec 层行为，不产生增量规范。）

  - `src/components/lcd/scenes/NearbyScene.vue` —— 与全程线路图保持版式一致、下一站圆点闪烁动画（Bug 2、3、5 视情况同步）
