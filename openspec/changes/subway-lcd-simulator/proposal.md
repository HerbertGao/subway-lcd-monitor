## 为什么

当前项目是一个空白项目，需要从零搭建一个基于 Web 的地铁 LCD 报站显示屏模拟器。真实的地铁 LCD 报站屏承载了线路图、到站信息、换乘指引等关键乘客信息，通过网页模拟器可以用于展示、教学、爱好者研究等场景，同时为未来接入真实数据或硬件显示奠定基础。

## 变更内容

- **新增**：建立完整的项目基础设施（Vue 3 + TypeScript + Vite + pnpm）
- **新增**：地铁线网数据模型与数据管理（线网、线路、站点、换乘关系）
- **新增**：列车运行状态机（停靠在站 → 关门出发 → 运行中 → 即将到站，循环推进）
- **新增**：画面轮播系统（同一列车状态下支持多个场景定时自动轮播或手动切换）
- **新增**：LCD 显示屏渲染引擎（全线路图、近几站放大图、换乘信息、到站提示等场景）
- **新增**：主题/皮肤系统（支持不同城市/线路混搭不同视觉风格，场景组件可被主题覆盖）
- **新增**：操作控制面板（线路选择、方向选择、列车运行控制、手动场景切换）
- **新增**：半自动数据采集脚本（辅助录入线网数据）

## 功能 (Capabilities)

### 新增功能

- `data-model`: 地铁线网数据模型 — 定义 Network、Line、Station 等核心数据结构，以及数据的加载、查询和校验
- `train-state-machine`: 列车运行状态机 — 管理列车在各站点间的运行状态流转（停靠、出发、运行、到站）
- `scene-rotation`: 画面轮播系统 — 在每个列车状态下管理多个显示场景的定时轮播与手动切换
- `lcd-renderer`: LCD 显示屏渲染 — 各类场景组件的渲染，包括全线路图(SVG)、近几站放大、换乘信息、到站提示等
- `theme-system`: 主题/皮肤系统 — 支持视觉样式与行为配置的主题定义，按城市/线路级别绑定，场景组件可被主题覆盖
- `control-panel`: 操作控制面板 — 用户交互界面，用于选择线路、控制列车运行、切换画面

### 修改功能

无（全新项目）

## 影响

- **项目结构**：从空白 TypeScript 项目重构为 Vue 3 + Vite 项目，现有 `src/index.ts` 和 `tsconfig.json` 将被替换
- **依赖项**：新增 Vue 3、Vite、Pinia、Vue Router 等核心依赖
- **构建系统**：从 `tsc` 切换为 Vite 构建
- **包管理**：使用 pnpm 管理依赖
- **数据文件**：新增 JSON 格式的线网数据文件（`src/data/` 目录）
