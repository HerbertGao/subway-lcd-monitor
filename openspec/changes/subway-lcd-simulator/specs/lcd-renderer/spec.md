## ADDED Requirements

### 需求：LCD 主容器

系统必须提供 LCD 显示屏主容器组件（LcdScreen），负责：
- 根据当前列车状态和场景轮播状态，渲染对应的场景组件
- 管理场景切换的过渡动画
- 将主题的 CSS 变量注入到容器作用域

#### 场景：渲染当前场景

- **当** 列车状态为 RUNNING 且当前轮播场景为"全线路图"
- **那么** LcdScreen 必须渲染 FullRouteScene 组件

#### 场景：主题样式注入

- **当** 当前线路绑定了特定主题
- **那么** LcdScreen 必须将该主题的 CSS 变量（背景色、前景色、字体等）注入到容器元素

### 需求：全线路图场景

系统必须提供全线路图场景组件（FullRouteScene），使用 SVG 渲染完整线路的所有站点，显示：
- 所有站点名称（中文）
- 站点标记（已过站、当前站、未过站使用不同样式区分）
- 站点间的连线（使用线路颜色）
- 换乘站的特殊标记（显示可换乘的线路）
- 运行方向指示

#### 场景：站点状态区分

- **当** 列车当前停靠在第 N 站
- **那么** 第 1 到 N-1 站必须显示为"已过站"样式，第 N 站必须显示为"当前站"样式，第 N+1 到末站必须显示为"未过站"样式

#### 场景：换乘站标记

- **当** 某站点的 transfers 列表非空
- **那么** 该站点必须显示换乘标记，包含可换乘线路的颜色或名称

### 需求：近几站场景

系统必须提供近几站放大场景组件（NearbyScene），显示当前站附近的几个站点（前后各若干站），以更大的字体和更详细的信息展示：
- 站点中英文名称
- 当前站高亮
- 运行方向箭头
- 各站点的换乘信息

#### 场景：显示范围

- **当** 渲染近几站场景
- **那么** 系统必须显示当前站、前方至少 2 站和后方至少 2 站（如站点不足则显示到线路端点）

#### 场景：到达起终点附近

- **当** 当前站距离线路起点或终点不足 2 站
- **那么** 系统必须显示到线路端点为止的所有站点，禁止显示空白占位

### 需求：换乘信息场景

系统必须提供换乘信息场景组件（TransferScene），当下一站或当前站为换乘站时，显示详细的换乘指引：
- 可换乘线路的名称和颜色
- 换乘线路的运行方向/终点站信息

#### 场景：下一站可换乘

- **当** 列车处于 RUNNING 状态且下一站为换乘站
- **那么** TransferScene 必须显示下一站可换乘的所有线路信息

#### 场景：无换乘信息

- **当** 当前站和下一站均无换乘线路
- **那么** TransferScene 可显示线路信息或其他默认内容

### 需求：到站提示场景

系统必须提供到站提示场景组件（ArrivalScene），在列车即将到站（ARRIVING 状态）时显示：
- 下一站站名（中英文，大字号）
- 开门方向提示（左侧开门/右侧开门/双侧开门）
- 如果下一站是换乘站，显示换乘提示

#### 场景：普通到站

- **当** 列车进入 ARRIVING 状态且下一站为非换乘站
- **那么** ArrivalScene 必须显示下一站中英文名称和开门方向

#### 场景：换乘站到站

- **当** 列车进入 ARRIVING 状态且下一站为换乘站
- **那么** ArrivalScene 必须同时显示站名、开门方向和可换乘线路信息

### 需求：LCD 外框装饰

系统必须提供 LCD 外框组件（LcdFrame），为 LCD 显示屏添加装饰性边框，模拟真实 LCD 屏幕的物理外观。外框样式由主题配置决定。

#### 场景：默认外框

- **当** 使用默认主题
- **那么** LcdFrame 必须渲染一个深色边框，模拟 LCD 屏幕的物理外壳
