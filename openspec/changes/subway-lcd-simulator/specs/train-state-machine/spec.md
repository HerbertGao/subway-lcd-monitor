## 新增需求

### 需求：列车运行状态定义

系统必须支持以下四种列车运行状态：
- `STOPPED` — 停靠在站（列车停在当前站）
- `DEPARTING` — 关门出发（列车即将离开当前站）
- `RUNNING` — 运行中（列车在两站之间行驶）
- `ARRIVING` — 即将到站（列车接近下一站）

#### 场景：初始状态

- **当** 选择一条线路并开始模拟
- **那么** 列车状态必须初始化为 STOPPED，当前站为该线路的第一个站点

### 需求：状态流转规则

列车状态必须按以下固定顺序单向流转：
STOPPED → DEPARTING → RUNNING → ARRIVING → STOPPED（下一站）

禁止跳过中间状态，禁止反向流转。

#### 场景：正常流转

- **当** 列车处于 STOPPED 状态并触发出发
- **那么** 状态必须依次经过 DEPARTING → RUNNING → ARRIVING → STOPPED

#### 场景：禁止跳跃

- **当** 列车处于 STOPPED 状态
- **那么** 系统禁止直接跳转到 RUNNING 或 ARRIVING 状态

### 需求：站点推进

每次状态从 ARRIVING 流转到 STOPPED 时，当前站索引必须前进一位（沿当前运行方向）。

#### 场景：到站推进

- **当** 列车从 ARRIVING 状态流转到 STOPPED
- **那么** 当前站必须更新为下一个站点

#### 场景：反向运行推进

- **当** 列车方向为反向（从终点往起点方向）且从 ARRIVING 流转到 STOPPED
- **那么** 当前站索引必须向起点方向前进一位

### 需求：终点站处理

当列车到达线路终点站时，系统必须根据线路类型处理：
- 非环线：自动反转运行方向
- 环线：继续沿同一方向循环到第一站

#### 场景：非环线到达终点

- **当** 列车到达非环线的终点站（最后一个站点）
- **那么** 系统必须自动反转运行方向，下一次出发时列车朝反方向运行

#### 场景：环线到达最后一站

- **当** 列车到达环线 stations 列表的最后一个站点
- **那么** 系统必须将下一站设为 stations 列表的第一个站点，方向不变

### 需求：运行方向

系统必须维护当前运行方向，支持两个方向：
- `FORWARD` — 沿 stations 列表正序方向运行
- `BACKWARD` — 沿 stations 列表逆序方向运行

用户必须能够在 STOPPED 状态下手动切换运行方向。

#### 场景：手动切换方向

- **当** 列车处于 STOPPED 状态且用户请求切换方向
- **那么** 运行方向必须翻转

#### 场景：运行中禁止切换方向

- **当** 列车处于 DEPARTING、RUNNING 或 ARRIVING 状态
- **那么** 系统禁止切换运行方向

### 需求：状态流转触发方式

状态流转必须支持两种触发方式：
- **手动模式**：用户通过操作面板按钮触发每一步状态变化
- **自动模式**：系统按预设的时间间隔自动推进状态流转

#### 场景：手动触发

- **当** 系统处于手动模式且用户点击"下一步"
- **那么** 列车状态必须前进到下一个状态

#### 场景：自动运行

- **当** 系统处于自动模式
- **那么** 系统必须按每个状态配置的持续时间自动推进到下一个状态
