## ADDED Requirements

### 需求：场景列表配置

每个列车运行状态必须关联一组可轮播的场景（Scene）列表。场景列表由主题配置定义，每个场景包含：
- `id`: 场景标识符（string）
- `duration`: 该场景的显示持续时间（秒）
- `component`: 可选的覆盖组件（用于主题定制）

#### 场景：读取场景配置

- **当** 列车进入某个运行状态
- **那么** 系统必须从当前主题配置中获取该状态对应的场景列表

#### 场景：单场景状态

- **当** 某个列车状态只配置了一个场景
- **那么** 系统必须持续显示该场景，不执行轮播

### 需求：定时自动轮播

当列车状态对应的场景列表包含多个场景时，系统必须按每个场景配置的 duration 定时自动切换到下一个场景。最后一个场景结束后必须循环回第一个场景。

#### 场景：正常轮播

- **当** 当前场景的 duration 时间到达
- **那么** 系统必须自动切换到场景列表中的下一个场景

#### 场景：循环轮播

- **当** 当前显示的是场景列表中的最后一个场景且其 duration 时间到达
- **那么** 系统必须循环回场景列表的第一个场景

### 需求：列车状态变化时重置轮播

当列车运行状态发生变化时，场景轮播必须重置：
- 切换到新状态对应的场景列表
- 从该列表的第一个场景开始显示
- 重置轮播计时器

#### 场景：状态变化重置

- **当** 列车状态从 RUNNING 变为 ARRIVING
- **那么** 系统必须立即切换到 ARRIVING 状态配置的第一个场景，并重新开始轮播计时

### 需求：手动切换场景

用户必须能够通过操作面板手动切换到当前状态的场景列表中的任意场景。手动切换后，轮播计时器必须重置为该场景的 duration。

#### 场景：手动选择场景

- **当** 用户通过操作面板选择当前状态下的某个特定场景
- **那么** 系统必须立即切换到该场景，并以该场景的 duration 重新开始计时

### 需求：场景切换动画

场景切换时必须支持过渡动画。动画类型由主题配置决定，默认使用淡入淡出（fade）效果。

#### 场景：带动画切换

- **当** 场景发生切换（自动轮播或手动切换）
- **那么** 系统必须使用主题配置的过渡动画完成场景切换，禁止直接无过渡跳变
