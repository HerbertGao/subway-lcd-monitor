## 新增需求

### 需求：线路选择

操作面板必须提供线路选择功能，允许用户：
- 选择城市
- 选择该城市下的具体线路
- 选择运行方向（起点→终点 或 终点→起点）

选择完成后，系统必须加载对应的线路数据和主题，并将列车初始化到第一站的 STOPPED 状态。

#### 场景：选择线路并初始化

- **当** 用户选择了城市"北京"、线路"燕房线"、方向"FORWARD"
- **那么** 系统必须加载燕房线数据和对应主题，列车状态设为 STOPPED，当前站设为燕房线第一站

#### 场景：切换线路

- **当** 列车正在运行中且用户选择了另一条线路
- **那么** 系统必须停止当前模拟，加载新线路数据，重新初始化列车状态

### 需求：列车运行控制

操作面板必须提供列车运行控制功能：
- **下一步**按钮：手动推进到下一个列车状态
- **自动/手动**切换：在自动运行和手动控制之间切换
- **方向切换**按钮：在列车 STOPPED 状态下切换运行方向（非 STOPPED 状态下禁用）

#### 场景：手动推进

- **当** 用户点击"下一步"按钮且列车处于 STOPPED 状态
- **那么** 列车状态必须推进到 DEPARTING

#### 场景：启动自动模式

- **当** 用户将模式切换为"自动"
- **那么** 系统必须开始按预设时间间隔自动推进列车状态

#### 场景：自动模式下方向按钮状态

- **当** 列车处于自动模式且状态不是 STOPPED
- **那么** 方向切换按钮必须显示为禁用状态

### 需求：场景手动切换

操作面板必须显示当前列车状态下的所有可用场景列表，允许用户点击切换到任意场景。当前正在显示的场景必须高亮标识。

#### 场景：显示场景列表

- **当** 列车处于 RUNNING 状态且该状态配置了 3 个场景
- **那么** 操作面板必须列出这 3 个场景的名称，并高亮当前正在显示的场景

#### 场景：点击切换场景

- **当** 用户在场景列表中点击某个非当前场景
- **那么** LCD 显示屏必须立即切换到该场景

### 需求：状态信息显示

操作面板必须显示当前模拟的关键状态信息：
- 当前线路名称和颜色
- 当前站名
- 当前列车状态（中文描述）
- 运行方向
- 运行模式（手动/自动）

#### 场景：状态实时更新

- **当** 列车状态发生变化（如从 RUNNING 变为 ARRIVING）
- **那么** 操作面板中的列车状态显示必须立即更新为"即将到站"
