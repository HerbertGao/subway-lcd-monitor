# mtr-train-states 规范

## 目的
mtr-train-states 能力定义 subway-lcd-monitor 港铁运行状态的动态呈现，让列车运行状态（停靠在站、关门出发、运行中、即将到站）驱动 LCD 视觉——状态化的站点圆点（含下一站闪烁）、随状态切换的站名特写展示站与黄色安全条文案、开门方向提示、线路图蓝色提示条内容，以及随状态切换的场景配置。
## 需求
### 需求:场景配置必须随列车运行状态切换

`default` 主题的场景配置必须使站名特写场景在 `STOPPED`、`DEPARTING`、`ARRIVING` 状态显示、线路图场景在 `RUNNING` 状态显示，且站名特写在前三个状态下必为该状态场景列表中必然渲染的场景（列表唯一项或首项），以保证站名特写的状态化文案可见。

#### 场景:停站/关门/到站显示站名特写
- **当** 列车处于 `STOPPED`、`DEPARTING` 或 `ARRIVING` 状态
- **那么** LCD 屏渲染站名特写场景

#### 场景:运行中显示线路图
- **当** 列车处于 `RUNNING` 状态
- **那么** LCD 屏渲染线路图场景

### 需求:站名特写必须显示与运行状态匹配的展示站

站名特写的展示站（站名、圆点、黄条、开门方向共用同一站）必须由列车运行状态决定：`ARRIVING` 显示正在到达的站（`nextStation`）、`STOPPED` 与 `DEPARTING` 显示当前停靠站（`currentStation`）。

#### 场景:即将到站
- **当** 列车处于 `ARRIVING` 状态
- **那么** 站名特写显示 `nextStation`（正在到达的站）

#### 场景:停靠在站或关门出发
- **当** 列车处于 `STOPPED` 或 `DEPARTING` 状态
- **那么** 站名特写显示 `currentStation`（当前停靠站），而非下一段旅程的后续站

### 需求:站名特写黄色安全条文案必须随运行状态变化

站名特写黄色安全条文案必须随列车 `TrainState` 切换，四个 `TrainState` 均有对应文案（含 `RUNNING` 兜底）。文案映射必须集中维护（主题数据或映射常量），不得在组件中硬编码。

#### 场景:停靠在站
- **当** 列车处于 `STOPPED` 状态且显示站名特写
- **那么** 黄色安全条显示「請小心空隙　Please mind the gap」

#### 场景:关门出发
- **当** 列车处于 `DEPARTING` 状态且显示站名特写
- **那么** 黄色安全条显示「請勿靠近車門　Please stand back from the doors」

#### 场景:即将到站
- **当** 列车处于 `ARRIVING` 状态且显示站名特写
- **那么** 黄色安全条显示「請小心空隙　Please mind the gap」

#### 场景:每个状态均有文案
- **当** 列车处于四个 `TrainState`（含 `RUNNING`）中的任一状态
- **那么** 黄条文案映射对该状态都有定义、不出现空文案

### 需求:站名特写必须显示开门方向提示

站名特写必须根据展示站的 `doorSide` 显示港铁式开门方向提示（中英双语，含视觉区分）；`doorSide` 的 `left`、`right`、`both` 三态与空值均须有明确处理。

#### 场景:右侧开门
- **当** 展示站 `doorSide` 为 `right`
- **那么** 显示对应的港铁式开门方向提示（如「請在這邊落車」）

#### 场景:左侧开门
- **当** 展示站 `doorSide` 为 `left`
- **那么** 显示对应的港铁式开门方向提示（如「請往另一邊落車」）

#### 场景:双侧开门
- **当** 展示站 `doorSide` 为 `both`
- **那么** 显示双侧开门的港铁式提示

#### 场景:开门方向数据缺失
- **当** 展示站无 `doorSide` 数据
- **那么** 不显示开门方向提示，站名特写其余内容正常渲染

### 需求:线路图站点圆点必须按列车运行状态着色

线路图场景的站点圆点必须由列车运行状态驱动着色：已过站为白色实心；未过站为黄色实心；列车正驶向的下一站（`nextStation`）圆点必须闪烁——在黄色实心与白色实心两态之间交替。下一站圆点的闪烁只改变填充色（黄↔白），其描边必须恒为深色、在闪烁全程保持不变，且闪烁两态均为实心填充、禁止出现空心圆环。圆点的填充色与描边色（含未过站黄色）必须经主题系统的 `ColorConfig` 与 CSS 变量提供。

#### 场景:渲染各状态站点圆点
- **当** 线路图渲染站点圆点
- **那么** 已过站圆点为白色实心、未过站圆点为黄色实心、下一站圆点在黄色实心与白色实心之间闪烁交替，且三类圆点的描边均为深色

#### 场景:下一站闪烁两态均为实心且描边恒定
- **当** 下一站圆点处于闪烁过程中（无论黄色态或白色态）
- **那么** 圆点均为实心填充（黄色或白色）、描边恒为深色不随闪烁改变，不出现空心圆环、不出现描边变色

### 需求:线路图底部蓝色提示条必须显示有意义的提示内容

线路图底部蓝色提示条必须显示有意义的港铁式提示内容，取代期一的固定占位文字。提示文案必须取自一组集中维护的预定义港铁式提示语（如安全、礼让、禁止饮食等主题的双语提示），不得在组件中硬编码、不得为空文案。

#### 场景:渲染线路图蓝条
- **当** 全程线路图或近段线路图场景渲染
- **那么** 底部蓝色提示条显示一条取自预定义提示语集的非空港铁式提示内容

### 需求:站名特写圆点必须按运行状态着色与闪烁

站名特写场景的圆点必须由列车运行状态驱动着色：当列车处于 `ARRIVING` 状态（站名特写展示正驶向的下一站）时，圆点必须闪烁——在黄色实心与白色实心之间交替、描边恒为深色不变，与线路图下一站圆点的闪烁规律一致；当列车处于 `STOPPED` 或 `DEPARTING` 状态（站名特写展示已停靠的当前站）时，圆点为白色实心、深色描边、不闪烁。圆点的填充色与描边色必须经主题系统的 `ColorConfig` 与 CSS 变量提供。

#### 场景:即将到站时圆点闪烁
- **当** 列车处于 `ARRIVING` 状态、站名特写展示正驶向的下一站
- **那么** 该圆点在黄色实心与白色实心之间闪烁交替、描边恒为深色不变，两态均为实心、不出现空心圆环

#### 场景:停靠或关门时圆点静止白实心
- **当** 列车处于 `STOPPED` 或 `DEPARTING` 状态、站名特写展示已停靠的当前站
- **那么** 该圆点为白色实心、深色描边、不闪烁
