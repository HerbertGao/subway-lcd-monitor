# mtr-visual-style 规范

## 目的
mtr-visual-style 能力定义 subway-lcd-monitor 的 LCD 报站屏港铁化视觉风格，涵盖港铁化配色（浅灰屏底、双色线路条、黄色安全条、蓝色提示条）、超宽横条比例、统一的白色实心站点圆点，以及三类场景（全程线路图、近段线路图、站名特写）的港铁版式。

## 需求
### 需求:LCD 屏必须采用港铁式超宽横条比例

LCD 屏必须采用约 7∶2（3.5∶1）的超宽横条比例，以贴合港铁车厢报站屏形态。比例调整必须沿用既有响应式机制——窄屏不溢出、不产生横向滚动，并以最小高度下限保证场景内容不被裁剪。

#### 场景:宽屏下显示
- **当** 视口宽度充足
- **那么** LCD 屏以约 7∶2 的超宽横条比例显示

#### 场景:窄屏下场景不被裁剪
- **当** 在视口宽度 320 至 1280 范围内显示任一场景
- **那么** LCD 屏整体（含外框）收缩适应可用宽度、不溢出、不产生横向滚动，且场景根元素内容不被裁剪

### 需求:LCD 屏不再渲染独立 header 与 footer

LCD 屏必须由当前场景整屏占满，不得再渲染独立的线路名 header 与运行状态 footer；线路名、方向、站名等信息由当前场景自身负责呈现。

#### 场景:渲染 LCD 屏
- **当** LCD 屏渲染任一场景
- **那么** 该场景占满整个 LCD 屏区域，屏上不存在独立的线路名 header 条与运行状态 footer 条

### 需求:LCD 屏必须采用港铁化配色且配色经主题系统提供

LCD 屏必须采用港铁风配色——浅灰屏底、灰色已过线路条（已过站名同步转灰）、未过线路条为线路自身色。港铁主题色板（屏底、已过段灰、黄色安全条、蓝色提示条、站点圆点色）必须经主题系统的 `ColorConfig` 提供并以 CSS 变量注入，禁止在组件中写死这些主题色值；未过线路条颜色由线路数据 `line.color` 驱动。

#### 场景:渲染 LCD 屏配色
- **当** LCD 屏渲染
- **那么** 屏底为浅灰、已过段线路条与已过站名为灰、未过段线路条为线路自身色，主题色取自注入的 CSS 变量、未在组件中硬编码

#### 场景:主题配色字段完备
- **当** 主题需要表达港铁专用配色（黄色安全条、蓝色提示条、站点圆点色）
- **那么** 这些配色由 `ColorConfig` 的字段承载、由 `useTheme` 注入为 CSS 变量供组件取用

### 需求:站点圆点采用统一样式

期一线路图场景中的站点圆点必须采用统一的白色实心样式（带描边），不按状态着色。已过站、当前站、未来站的区分由线路条颜色（灰 / 线路自身色）、站名颜色（已过转灰）与当前站处的行进方向箭头共同表达。

#### 场景:渲染站点圆点
- **当** 线路图渲染站点圆点
- **那么** 各站点圆点为统一的白色实心样式，已过/未过的区分体现在线路条颜色与站名颜色上、不体现在圆点颜色上

### 需求:站名特写场景必须采用港铁版式

到站提示场景必须采用港铁站名特写版式——上半浅灰区横向排布超大中文站名、圆点与英文名，下半为黄色安全条。

#### 场景:渲染站名特写
- **当** 到站提示场景渲染
- **那么** 上半区横向排布超大中文站名、白色实心圆点与英文站名，下半区为黄色安全条

### 需求:线路图场景必须采用港铁版式并含底部提示条

全程线路图与近段线路图场景必须采用港铁版式——站点横排、每站圆点上方自上而下排列中文名与英文名（中英文均在圆点上方、中文名在英文名之上）、双色线路条贯穿（已过灰 / 未过为线路自身色）、当前站处显示行进方向箭头，并在底部渲染蓝色提示条。

#### 场景:渲染全程线路图
- **当** 全程线路图场景渲染
- **那么** 全部站点横向排列、每站圆点上方自上而下依次为中文名与英文名、线路条按已过/未过着灰/线路自身色、当前站处有行进方向箭头、底部有蓝色提示条

#### 场景:渲染近段线路图
- **当** 近段线路图场景渲染
- **那么** 以与全程线路图一致的港铁视觉语言显示当前站附近的数个站点、字号更大、底部含蓝色提示条

### 需求:期一不渲染换乘线路标签

期一所有港铁场景（全程线路图、近段线路图、站名特写）禁止渲染换乘线路标签 / badge——换乘线路标签展示属后续阶段；现有场景中已有的换乘 badge 必须在重绘时移除。

#### 场景:站点含换乘数据
- **当** 某站点数据包含换乘线路（`transfers` 非空）并在任一港铁场景中渲染
- **那么** 该场景不渲染任何换乘线路标签 / badge
