# mtr-visual-style 规范

## 目的
mtr-visual-style 能力定义 subway-lcd-monitor 的 LCD 报站屏港铁化视觉风格，涵盖港铁化配色（浅灰屏底、双色线路条、黄色安全条、蓝色提示条）、超宽横条比例、站点圆点的统一基础几何形态，以及三类场景（全程线路图、近段线路图、站名特写）的港铁版式。站点圆点随列车运行状态变化的着色由 `mtr-train-states` 能力定义。
## 需求
### 需求:LCD 屏必须采用港铁式超宽横条比例

LCD 屏必须采用约 7∶2（3.5∶1）的超宽横条比例，以贴合港铁车厢报站屏形态。比例调整必须沿用既有响应式机制——窄屏不溢出、不产生横向滚动，并以最小高度下限保证场景内容不被裁剪。

#### 场景:宽屏下显示
- **当** 视口宽度充足
- **那么** LCD 屏以约 7∶2 的超宽横条比例显示

#### 场景:窄屏下场景不被裁剪
- **当** 在视口宽度 320 至 1280 范围内显示任一场景
- **那么** LCD 屏整体（含外框）收缩适应可用宽度、不溢出、不产生横向滚动，且场景根元素内容不被裁剪

### 需求:LCD 屏不再渲染独立 header 与 footer

LCD 屏必须由当前场景整屏占满，不得再渲染独立的线路名 header 与运行状态 footer；线路名、方向、站名等信息由当前场景自身负责呈现。

#### 场景:渲染 LCD 屏
- **当** LCD 屏渲染任一场景
- **那么** 该场景占满整个 LCD 屏区域，屏上不存在独立的线路名 header 条与运行状态 footer 条

### 需求:LCD 屏必须采用港铁化配色且配色经主题系统提供

LCD 屏必须采用港铁风配色——浅灰屏底、灰色已过线路条（已过站名同步转灰）、未过线路条为线路自身色。港铁主题色板（屏底、已过段灰、黄色安全条、蓝色提示条、站点圆点色）必须经主题系统的 `ColorConfig` 提供并以 CSS 变量注入，禁止在组件中写死这些主题色值；未过线路条颜色由线路数据 `line.color` 驱动。

#### 场景:渲染 LCD 屏配色
- **当** LCD 屏渲染
- **那么** 屏底为浅灰、已过段线路条与已过站名为灰、未过段线路条为线路自身色，主题色取自注入的 CSS 变量、未在组件中硬编码

#### 场景:主题配色字段完备
- **当** 主题需要表达港铁专用配色（黄色安全条、蓝色提示条、站点圆点色）
- **那么** 这些配色由 `ColorConfig` 的字段承载、由 `useTheme` 注入为 CSS 变量供组件取用

### 需求:站点圆点采用统一样式

线路图场景中的站点圆点必须采用统一的圆形标记作为基础几何形态——所有站点圆点共用一致的基础半径与描边粗细；当前站圆点可在该基础半径上略作放大以突出，属允许的强调手段。站点圆点的着色（填充色与描边色）由 `mtr-train-states` 能力按列车运行状态定义；`mtr-visual-style` 仅定义圆点的基础几何形态，不约束其颜色。

#### 场景:站点圆点基础形态
- **当** 线路图渲染站点圆点
- **那么** 各站点圆点为统一的圆形标记（共用一致的基础半径与描边粗细，当前站圆点可略放大以突出），其填充色与描边色由 `mtr-train-states` 能力按列车运行状态定义

### 需求:站名特写场景必须采用港铁版式

到站提示场景必须采用港铁站名特写版式——上半浅灰区横向排布超大中文站名、圆点与英文名，下半为黄色安全条。圆点为统一的圆形实心标记、深色描边；其随列车运行状态的着色与闪烁由 `mtr-train-states` 能力定义，`mtr-visual-style` 不约束其填充色。

黄色安全条必须随其文案呈现港铁式版式差异：当黄条文案为「請小心空隙」类（停站 / 到站）时，黄条两端各渲染一个港铁式警示图标（红圈警示标，以纯 SVG / CSS 绘制、不引入图片资源）；当文案为「請勿靠近車門」类（关门发车）时，黄条不渲染两端图标。黄条版式（如高度）可随文案微调。

#### 场景:渲染站名特写
- **当** 到站提示场景渲染
- **那么** 上半区横向排布超大中文站名、实心圆点与英文站名，下半区为黄色安全条；圆点为深色描边的圆形实心标记，其状态化着色由 `mtr-train-states` 定义

#### 场景:停站或到站黄条带警示图标
- **当** 站名特写黄条文案为「請小心空隙」类
- **那么** 黄条两端各渲染一个港铁式红圈警示图标

#### 场景:关门发车黄条不带图标
- **当** 站名特写黄条文案为「請勿靠近車門」类
- **那么** 黄条不渲染两端警示图标

### 需求:线路图场景必须采用港铁版式并含底部提示条

全程线路图与近段线路图场景必须采用港铁版式——站点横排、每站圆点上方自上而下排列中文名与英文名（中英文均在圆点上方、中文名在英文名之上）、双色线路条贯穿（已过灰 / 未过为线路自身色）、显示行进方向箭头（行进方向箭头的位置与随运行状态的闪烁由 `mtr-train-states` 能力定义），并在底部渲染蓝色提示条。

线路图场景必须填满整个 LCD 屏区域，底部蓝色提示条必须贴 LCD 屏底边，不得使提示条悬在屏中部、其下残留大片空白。

站名在站点横向排布的间距内必须自适应、不得相互重叠——当线路站点较多、站点横向间距收窄时，站名（尤其较长的英文名）必须随之收缩或压缩，以保证相邻站站名不重叠且仍可读。

#### 场景:渲染全程线路图
- **当** 全程线路图场景渲染
- **那么** 全部站点横向排列、每站圆点上方自上而下依次为中文名与英文名、线路条按已过/未过着灰/线路自身色、线路上有行进方向箭头（位置与闪烁由 `mtr-train-states` 定义）、底部有蓝色提示条

#### 场景:渲染近段线路图
- **当** 近段线路图场景渲染
- **那么** 以与全程线路图一致的港铁视觉语言显示当前站附近的数个站点、字号更大、底部含蓝色提示条

#### 场景:线路图填满 LCD 屏且蓝条贴底
- **当** 全程线路图或近段线路图场景渲染，且无论该线路站点多少
- **那么** 该场景填满整个 LCD 屏区域、底部蓝色提示条贴 LCD 屏底边，屏内不残留大片空白

#### 场景:密集线路下站名不重叠
- **当** 线路站点较多致站点横向间距收窄并渲染线路图场景
- **那么** 各站中文名与英文名在其所属站间距内自适应收缩 / 压缩，相邻站站名不相互重叠、仍保持可读

### 需求:线路图场景换乘站必须渲染换乘线路标签

全程线路图与近段线路图场景中，含换乘线路（`transfers` 非空）的站点必须在其站点圆点附近渲染换乘线路标签——港铁式彩色线路名牌，采用换乘线路自身色，并标注该换乘线路的名称。换乘标签的颜色与文案必须由线路数据（`transfers` 中的 `lineColor`、`lineName` 等字段）驱动，不在组件中写死。无换乘的站点不渲染换乘标签。

#### 场景:换乘站渲染换乘标签
- **当** 线路图场景渲染一个 `transfers` 非空的站点
- **那么** 该站点处渲染换乘线路标签，标签采用换乘线路自身色、标注换乘线路名称，颜色与文案取自 `transfers` 数据

#### 场景:无换乘站不渲染标签
- **当** 线路图场景渲染一个 `transfers` 为空的站点
- **那么** 该站点不渲染任何换乘线路标签

### 需求:全程线路图下一站为换乘站时必须渲染换乘线路支线

全程线路图必须在列车下一站为换乘站时，于该换乘站处渲染换乘线路「支线」——把可换乘的线路作为一条该线路自身色的短线路条（含其站点）画出，并以换乘节点与当前线路的换乘站连接，使换乘网络信息直接融入全程线路图。同时全程线路图底部蓝色提示条须切换为换乘提示文案（中英双语，如「往房山线各站，请在此站换乘」）。

「下一站为换乘站」指列车处于 `RUNNING` 状态、`nextStation` 的 `transfers` 含至少一个可在已加载线路中解析到的换乘线路。下一站非换乘站、或换乘线路均无法解析时，全程线路图照常渲染、不绘制换乘支线，底部蓝条为普通线路图提示语。换乘支线必须由换乘数据（`transfers` + 已加载线路）驱动构建，不写死特定线路。

不新增独立的「全网换乘图」场景——换乘网络信息融入全程线路图本身。

#### 场景:下一站为换乘站渲染换乘支线
- **当** 列车处于 `RUNNING` 状态、下一站为含可解析换乘线路的换乘站，全程线路图渲染
- **那么** 全程线路图在该换乘站处渲染换乘线路支线（换乘线路自身色短线路条 + 站点 + 换乘节点连接），底部蓝条显示换乘提示文案

#### 场景:下一站非换乘站不渲染支线
- **当** 列车下一站不是换乘站、或下一站换乘线路均无法解析，全程线路图渲染
- **那么** 全程线路图不渲染换乘支线，底部蓝条显示普通线路图提示语
